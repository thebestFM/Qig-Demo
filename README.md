## 📖 项目简介

**岐黄问道**是一个基于Vue 3开发的AI中医问诊Demo应用,旨在通过现代化的Web技术展现传统中医的智慧。应用采用"宣纸留白、朱砂点睛"的中国风设计理念,融合AI技术,为用户提供个性化的中医问诊体验。

### ✨ 核心特色

- 🎨 **中国风设计** - 宣纸白背景、朱砂红强调、书法字体点缀
- 🌿 **四诊合参** - 望闻问切完整的中医问诊流程
- 🤖 **AI赋能** - 智能分析、个性化建议、RAG引用
- 📱 **响应式设计** - 完美适配移动端和桌面端
- 🎯 **Demo可运行** - 所有功能可点击可体验,Mock数据完善

## 🎯 功能介绍

### 1. 今日 (Dashboard)

**核心功能:**
- 🌸 节气胶囊 - 根据当前日期显示节气信息
- 📊 健康数据卡片 - 恢复指数、情绪、体质趋势
- ✅ 今日行动 - 可勾选的任务列表
- 📜 经典知识库 - 每日一句经典引用
- 🚀 快速入口 - 一键开始四诊合参

**Mock数据:**
- 睡眠时长、HRV、心率、步数
- 7天体质趋势图
- 今日行动项列表

### 2. 问诊 (AI问诊流程)

**流程状态机:**
```
空闲 → 采集 → 问卷 → 确认 → 生成 → 报告
```

**采集模块:**
- 📸 **舌象上传** - 图片上传 + Mock质量评估
- 🤳 **面象上传** - 图片预览
- 🎤 **声诊上传** - 音频文件上传占位
- 💓 **脉象选择** - 手动选择(快/缓、浮/沉)

**智能问卷:**
1. 入睡情况
2. 食欲与口渴
3. 大便小便情况
4. 情绪状态
5. 主要不适

**报告页内容:**
- 📋 本次概览
- 🏷️ 可能倾向标签
- 💡 今日建议(饮食/作息/穴位/运动)
- ⚠️ 红旗提示(安全守门)
- 📚 依据与引用(RAG证据卡)

### 3. 方子 (处方解析)

**功能:**
- 📸 **拍方识方** - 上传处方图片 → Mock OCR识别
- 🔍 **方子详情**
  - 核心思路总结
  - 逐味解析(性味归经/功效/注意事项)
  - 风险提示
  - RAG引用来源

**特色:**
- 每味药独立卡片展示
- 清晰的注意事项提醒
- 一键加入计划功能

### 4. 陪伴 (AI管家)

**核心功能:**
- 💬 **对话式交互** - 气泡式聊天界面
- 📊 **今日计划进度** - 实时显示完成度
- 🎯 **情境快捷入口** - 一键选择常见问题
  - 我睡不着 🌙
  - 我上火 🔥
  - 我没胃口 🍽️
  - 我很焦虑 😰
  - 我想运动 🏃

**AI建议:**
- 个性化健康建议
- 证据依据展示
- 一键加入计划

### 5. 医生 (匹配推荐)

**功能:**
- 👨‍⚕️ **智能匹配** - 根据问诊结果推荐医生
- 📊 **匹配度展示** - 百分比 + 匹配理由
- 🟢 **在线状态** - 实时显示医生状态
- 💰 **价格透明** - 问诊费用、等待时间
- ✅ **AI初筛报告** - 自动附带到问诊

**Demo特性:**
- 医生详情弹窗
- "立即问诊"Demo提示
- 跳转到陪伴Tab模拟医生助理接入

## 🚀 快速开始

### 环境要求

- Node.js >= 16.0.0
- npm >= 7.0.0

### 安装依赖

```bash
cd qihuang-consultation
npm install
```

### 启动开发服务器

```bash
npm run dev
```

### 构建生产版本

```bash
npm run build
```

### 预览生产版本

```bash
npm run preview
```

## 🎨 设计系统

### 配色方案

| 名称 | 色值 | 用途 |
|------|------|------|
| 宣纸白 | `#FAF8F3` | 主背景色 |
| 米白 | `#F5F3ED` | 次级背景 |
| 近墨黑 | `#2C2416` | 主文字色 |
| 朱砂红 | `#C84B31` | 强调色 |
| 竹青 | `#5B8C5A` | 成功色 |
| 金色 | `#E6A23C` | 警告色 |

### 字体系统

- **主标题**: 思源宋体 / Noto Serif SC
- **正文**: 思源黑体 / Noto Sans SC
- **副标题**: 楷体 / KaiTi

### 设计理念

1. **宣纸留白** - 充足的空白空间,营造宁静氛围
2. **朱砂点睛** - 重点信息使用朱砂红强调
3. **书法点缀** - 标题使用书法字体,增添文化韵味
4. **克制动效** - 简洁的过渡动画,避免喧宾夺主

## 📝 Mock数据说明

本项目为Demo演示,所有数据均为Mock数据,不涉及真实医疗诊断:

- 健康数据(睡眠、HRV、心率等)为随机生成
- 医生信息为虚构占位数据
- AI分析结果为预设模板
- OCR识别为模拟延迟

**⚠️ 重要提示**: 本Demo仅供技术演示,不可用于真实医疗诊断!

## 🔧 开发说明

### 代码规范

- 使用Vue 3 Composition API
- 组件采用`<script setup>`语法
- 样式使用Scoped CSS
- 遵循ESLint标准规范

### 状态管理

每个模块独立store:
- `user.js` - 用户健康数据
- `consultation.js` - 问诊流程状态
- `prescription.js` - 处方数据
- `companion.js` - 对话历史
- `doctor.js` - 医生列表

### 路由配置

采用底部Tab导航模式,5个主要路由:
- `/today` - 今日
- `/consultation` - 问诊
- `/prescription` - 方子
- `/companion` - 陪伴
- `/doctors` - 医生
